## Where full-size URLs come from
When you open an Instagram post in a browser, the page (or its background requests) includes structured data that contains direct CDN URLs for the media.  In that data, the first image can be found as `display_url`, and carousel posts include an `edge_sidecar_to_children` list with one “node” per slide. 

## How it handles multiple images
For each carousel “node,” Instagram provides multiple candidate image URLs (often shown as repeated `src` entries), and you can pick the highest-resolution one (commonly the last entry; Rooster’s walkthrough calls this the “full 1080 × 1080” image).  So an extension can iterate the carousel nodes and store one chosen URL per slide (or download all of them). 

## The “?__a=1” trick (and why it’s flaky)
A commonly described method is appending `?__a=1` to a post URL to expose the JSON that contains `display_url` and the carousel children list.  However, Instagram frequently changes gating/policies, so this may require being logged in, may intermittently fail, or may return blocked/login-wall behavior.
